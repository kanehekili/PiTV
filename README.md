# PiTV

Turn a Raspberry Pi 5 into a silent, headless streaming appliance — controlled entirely from your phone or tablet. No keyboard, no mouse, no remote control needed. Just power it on, pick a service, and watch.

## What it does

PiTV boots straight into a fullscreen Chromium kiosk. A lightweight Flask server runs in the background, serving a touch-optimised remote control UI to any device on your network. Open `http://pitv:5000` on your phone and you have full control — tap a tile to launch Netflix, YouTube, ARD or ZDF, swipe the touchpad to move the cursor, tap L/R to click, scroll, type, and power off the Pi when done.

No cloud, no account, no app to install. It's just a web page.

## Features

- **Kiosk mode** — Chromium starts fullscreen with no address bar, no tabs, no distractions
- **Touch remote** — relative touchpad with visual cursor feedback, left/right click, scroll up/down
- **Service launcher** — configurable tiles loaded from `pitv.ini`, each with a custom icon
- **Keyboard input** — type text directly into the browser from your phone
- **Volume control** — speaker up/down via PulseAudio
- **Safe shutdown** — two-tap power button with armed state and pulse animation
- **DRM streaming** — Netflix, Amazon Prime Video, Disney+, Apple TV+ and others work via Widevine (`libwidevinecdm0`)
- **Plymouth boot splash** — custom splash screen from power-on, no kernel text visible
- **Fully automated setup** — one Ansible playbook provisions a fresh Raspberry Pi OS install from scratch

## Hardware

- Raspberry Pi 4 or newer
- HDMI display / TV
- SD card (16 GB or more)
- Any phone or tablet as the remote

## Stack

**Chromium** runs in kiosk mode with the Chrome DevTools Protocol enabled — navigation commands are sent directly to the browser over a local WebSocket, no address bar needed.

**Flask** serves the remote control UI to your phone and handles all control requests — mouse movement, clicks, scrolling, keyboard input, volume and navigation.

**xdotool** translates Flask commands into real X11 input events — cursor movement, clicks and keystrokes land exactly where they should.

**PulseAudio** manages HDMI audio. The Ansible playbook auto-detects the correct sink at setup time and configures it as the default.

**Plymouth** takes over the framebuffer at boot and holds the splash screen until Chromium is ready — no kernel text, no login prompt, no flicker.

**Ansible** provisions everything in a single playbook run — packages, autologin, X session, Plymouth theme, audio config and app deployment.

## Installation

Flash a fresh **Raspberry Pi OS Lite 64-bit (Bookworm)** image, enable SSH, set the hostname to `pitv`.

On your machine, accept the host key:
```bash
ssh-keyscan -H pitv >> ~/.ssh/known_hosts
```

Run the playbook:
```bash
ansible-playbook -i ansible/inventory.ini ansible/pitv.yml
```

Enter the Pi password when prompted. Ansible will:
1. Update and upgrade all packages
2. Install X, Chromium, Flask, xdotool, PulseAudio, Plymouth and Widevine
3. Configure passwordless autologin
4. Deploy all app files
5. Set up Plymouth boot splash
6. Configure the HDMI audio sink automatically
7. Suppress boot messages

Reboot and you're done.

## Configuring services

Edit `src/pitv.ini` to add or change tiles:

```ini
[Netflix]
url = https://www.netflix.com
icon = netflix.png

[YouTube]
url = https://www.youtube.com
icon = youtube.png
```

Drop matching icons into `src/icons/` and re-run the playbook.

## Using the remote

Open `http://pitv:5000` on your phone or tablet.

- **Tap a tile** — launches the service in Chromium
- **Swipe the touchpad** — moves the cursor
- **L / R** — left and right click
- **↑ / ↓** — scroll up and down
- **🔊 / 🔉** — volume up and down
- **Type field** — sends text directly to the browser
- **⏎** — sends Enter key
- **Power button** — tap once to arm (red pulse), tap again within 3 seconds to shut down
- **Home button** — returns to the service launcher
